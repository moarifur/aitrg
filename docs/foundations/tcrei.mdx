# The TCREI Framework — Prompt Mastery

A practical system for writing AI prompts that get precise, useful results every time.

---

## What is TCREI?

TCREI is a five-element framework. Every great prompt uses at least two or three of these elements — the more complex the task, the more elements you need.

| Letter | Element | What it does |
|--------|---------|-------------|
| **T** | **Task** | The exact action you want performed. Be specific: *"write a 150-word intro"* beats *"write something"* |
| **C** | **Context** | The situation, audience, constraints, or background. This separates a generic answer from one that fits your world |
| **R** | **Role / Reference** | A persona or expert identity to adopt. Shapes tone, vocabulary, and angle |
| **E** | **Examples / Evaluations** | A sample of good (or bad) output. The fastest way to calibrate tone, style, or format |
| **I** | **Instructions / Itertions** | Format rules, constraints, things to avoid, or step-by-step requirements |

:::tip The single most common mistake
Most people write **Task-only** prompts and leave out Context. Adding just one sentence of context — audience, purpose, constraints — typically doubles the quality of the output.
:::

---

## Framework Quick Reference

```
T — Task        → What do you want done? (specific verb + deliverable)
C — Context     → Who is the audience? What's the situation?
R — Role        → What expert should Claude be?
E — Examples    → Show a sample of the output style or format
I — Instructions → What constraints, format rules, or things to avoid?
```

## Key Takeaways

:::note Rule 1 — Task is not enough
Almost every weak prompt above is just a Task. Adding Context alone — one sentence about your audience or situation — usually doubles output quality.
:::

:::note Rule 2 — Role changes everything
When you need expert-level output (code review, career advice, design critique), assigning a Role tells the model what vocabulary, assumptions, and standards to apply.
:::

:::note Rule 3 — Examples beat descriptions
You can describe tone for 5 sentences, or you can show one example. The example wins every time. Use Examples (E) whenever you care about style, format, or voice.
:::

:::note Rule 4 — Instructions are guardrails
The most common Instructions are: length limits, what to exclude, format specs, and things *not* to do. These prevent the most common failure modes.
:::

:::tip Practice exercise
Take any prompt you've used in the past week. Identify which TCREI elements it has. Now add the missing ones. Run both versions and compare.
:::

## TCREI Cheat Sheet

```
┌─────────────────────────────────────────────────────────┐
│                  TCREI PROMPT FORMULA                   │
├────────┬────────────┬──────────────────────────────────┤
│ T      │ Task       │ Specific verb + deliverable       │
│ C      │ Context    │ Audience, situation, constraints  │
│ R      │ Role       │ "Act as a [expert]…"              │
│ E      │ Examples   │ Show one good or bad sample       │
│ I      │ Instructions│ Format, length, what to avoid   │
└────────┴────────────┴──────────────────────────────────┘

You don't need all 5 every time.
  Simple tasks     → T + I (2 elements)
  Expert output    → T + C + R (3 elements)
  Style-sensitive  → T + C + E + I (4 elements)
  Complex briefing → All 5
```
